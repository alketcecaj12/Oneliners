# Python Oneliners

A growing collection of Jupyter notebooks demonstrating the power of **single-line Python** — from core language idioms to data analysis, NumPy array operations, regex, and machine learning.

Each notebook is self-contained and paired with a **data generator script** that produces realistic, finance-flavoured synthetic datasets so every oneliner has something meaningful to run on.

---

## Why Oneliners?

- **Forces clarity** — when every character counts, there is no room for unfocused code
- **Deepens Python fluency** — list comprehensions, lambdas, `map`/`filter`, walrus operator, unpacking
- **Teaches idiomatic thinking** — the same logic a senior engineer reaches for instinctively
- **Impresses in interviews** — concise, correct, readable solutions stand out
- **Transfers directly to production** — pandas, NumPy, and sklearn oneliners appear constantly in real data pipelines

### `Oneliners_2` — Pandas Data Analysis

Applied to a **synthetic bank customer portfolio** (1,000 rows, 13 columns) generated by `Oneliners_2_data_generator.py`.

#### Dataset

| Column | Type | Description |
|---|---|---|
| `customer_id` | str | Unique identifier |
| `segment` | cat | Retail / Corporate / SME / Private Banking |
| `region` | cat | Copenhagen / Aarhus / Odense / Aalborg |
| `product` | cat | Mortgage / Consumer Loan / Credit Card / Overdraft |
| `collateral` | cat | Real Estate / Financial Asset / Unsecured |
| `rating` | cat | AAA → CCC credit rating |
| `age` | int | Customer age |
| `income` | float | Annual income (DKK), log-normal |
| `loan_amount` | float | Exposure (DKK), log-normal |
| `LTV` | float | Loan-to-Value ratio, beta-distributed |
| `months_active` | int | Relationship duration |
| `revenue` | float | Bank revenue from customer (heteroskedastic) |
| `default` | int | Default flag — ~12% positive rate |

Missings injected at 4–7% on `income`, `LTV`, `collateral`, `rating`.

#### Oneliners covered

```python
# Full missing value report as percentage, sorted
df.isnull().mean().mul(100).round(2).sort_values(ascending=False)

# Correlation of all numeric columns with the target
df.corr()['default'].drop('default').sort_values(key=abs, ascending=False)

# Value counts as % with cumulative share
df['rating'].value_counts(normalize=True).mul(100).cumsum().round(2)

# Group stats — mean, std, count, median in one shot
df.groupby('segment')['revenue'].agg(['mean', 'std', 'count', 'median']).round(2)

# Cross-tab with normalised row percentages
pd.crosstab(df['default'], df['collateral'], normalize='index').mul(100).round(1)

# Winsorise: clip to 1st–99th percentile
df['LTV'] = df['LTV'].clip(*df['LTV'].quantile([0.01, 0.99]))

# Select only numeric columns and describe, sorted by std
df.select_dtypes(include='number').describe().T.sort_values('std', ascending=False)

# Flatten MultiIndex after groupby
df.groupby(['region', 'product'])['revenue'].sum().reset_index()
```

### `Oneliners_3` — NumPy Array Operations

Applied to **six purpose-built arrays** generated by `Oneliners_3_data_generator.py`.

#### Arrays

| Variable | Shape | Description |
|---|---|---|
| `X` | (1000, 5) | Loan feature matrix — `loan_amount`, `LTV`, `income`, `months_active`, `recovery_rate` |
| `monthly_losses` | (60,) | Monthly portfolio loss rates with sinusoidal macro cycle |
| `returns` | (120, 5) | 10-year monthly asset returns — correlated 5-asset portfolio |
| `credit_scores` | (1000,) | Credit scores with 30 outliers injected at both extremes |
| `LTV_arr` | (1000,) | LTV column extracted for threshold operations |
| `X_small` | (10, 2) | 10-loan slice for readable pairwise distance output |

#### Oneliners covered

```python
# Normalise to [0, 1]
monthly_losses_norm = (monthly_losses - monthly_losses.min()) / (monthly_losses.max() - monthly_losses.min())

# Z-score standardise
monthly_losses_z = (monthly_losses - monthly_losses.mean()) / monthly_losses.std()

# 7-month rolling mean (smoothing the macro cycle)
rolling_mean = np.convolve(monthly_losses, np.ones(7) / 7, mode='valid')

# Pairwise Euclidean distances between 10 loans
distances = np.sqrt(((X_small[:, None] - X_small[None, :]) ** 2).sum(-1))

# Count loans above LTV threshold
(LTV_arr > 0.8).sum()

# Asset return covariance matrix — upper triangle only
np.triu(np.cov(returns.T), k=1)

# Column-wise min-max normalisation of the full feature matrix
X_norm = (X - X.min(axis=0)) / (X.max(axis=0) - X.min(axis=0))

# IQR outlier count in credit scores
q1, q3 = np.percentile(credit_scores, [25, 75])
(credit_scores < q1 - 1.5*(q3-q1)).sum() + (credit_scores > q3 + 1.5*(q3-q1)).sum()

# Winsorise credit scores to IQR bounds
credit_scores_clipped = np.clip(credit_scores, q1 - 1.5*(q3-q1), q3 + 1.5*(q3-q1))
```

## Design Principles

Every oneliner in this repo satisfies three conditions:

1. **Fits on one line** — no semicolon-chaining of unrelated statements
2. **Readable** — another Python programmer can understand it without comments
3. **Useful** — solves a real data analysis problem, not a contrived puzzle
